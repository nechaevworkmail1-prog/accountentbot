# Рекомендации по хостингу для Accountant Bot

## Анализ требований бота

### Технические характеристики:
- **Язык/Платформа**: Java 17 (Eclipse Temurin)
- **Архитектура**: Telegram Long Polling Bot
- **Хранилище данных**: 
  - CSV файлы (локальное хранилище)
  - Google Sheets API (опционально)
- **Зависимости**: 
  - Telegram Bot API
  - Google Sheets API
  - JFreeChart (генерация графиков)
- **Особенности**:
  - Health check HTTP сервер (порт из переменной окружения PORT)
  - Планировщики задач (recurring expenses, monthly reports)
  - Требуется постоянное хранилище для CSV файлов
  - Нужен 24/7 uptime для работы бота

### Ресурсы:
- **RAM**: Минимум 512MB (указано в Dockerfile: `-Xmx512m`)
- **CPU**: Низкая нагрузка (Telegram бот)
- **Диск**: Небольшой объем (CSV файлы, но нужна персистентность)
- **Сеть**: Исходящие HTTPS запросы к Telegram API и Google Sheets API

## Итоговая рекомендация

### 🏆 **Лучший выбор: Railway** (если Render не подошел)

**Почему Railway**:
1. Идеальный баланс цены и функциональности
2. Отличная поддержка Docker (ваш проект уже использует Dockerfile)
3. Простая настройка постоянного хранилища
4. Автоматический деплой из Git
5. Бесплатный пробный период для тестирования
6. Health checks работают из коробки
7. Меньше проблем с cold starts, чем на Render

### 📝 **Если уже пробовали Render:**

**Если Render работал, но были проблемы:**
- Проверьте настройку Render Disk (см. `RENDER_SETUP.md`)
- Убедитесь, что используете Starter план (не Free)
- Проверьте переменные окружения

**Если Render не подошел по другим причинам:**
- **Railway** - лучшая альтернатива, проще настройка
- **Fly.io** - если нужна лучшая производительность
- **VPS** - если нужен полный контроль и минимальная цена

### 📋 План миграции на Railway:

1. **Создать аккаунт на Railway** (railway.app)
2. **Подключить GitHub репозиторий**
3. **Настроить переменные окружения**:
   - `BOT_TOKEN` - токен Telegram бота
   - `BOT_USERNAME` - username бота
   - `GOOGLE_CREDENTIALS` - JSON credentials для Google Sheets (или путь к файлу)
   - `GOOGLE_SPREADSHEET_ID` - ID таблицы (опционально)
   - `DATA_DIR` - путь к директории данных (Railway автоматически создаст volume)
   - `PORT` - порт для health check (Railway установит автоматически)

4. **Настроить Persistent Volume** для директории `data/`
5. **Деплой и тестирование**

## Важные замечания

1. **Постоянное хранилище критично** - CSV файлы должны сохраняться между перезапусками
2. **Health checks** - ваш бот уже имеет health check endpoint, это хорошо
3. **Переменные окружения** - все секреты должны быть в переменных окружения, не в коде
4. **Google Sheets credentials** - можно хранить как переменную окружения `GOOGLE_CREDENTIALS` (JSON строка) или файл
5. **Мониторинг** - рассмотрите добавление логирования и мониторинга (например, Sentry)

## Следующие шаги

### Если используете Render:
1. См. подробную инструкцию в `RENDER_SETUP.md`
2. Убедитесь, что настроен Render Disk
3. Проверьте переменные окружения
4. Используйте Starter план для 24/7 работы

### Если пробуете другой хост:
1. Выберите хост (рекомендую Railway, если Render не подошел)
2. Создайте аккаунт и проект
3. Подключите репозиторий
4. Настройте переменные окружения (те же, что для Render)
5. Настройте persistent volume для `data/`
6. Задеплойте и протестируйте
7. Настройте мониторинг (опционально)

**Примечание**: Код уже адаптирован под Render, но будет работать на любом Docker-хосте с минимальными изменениями.
