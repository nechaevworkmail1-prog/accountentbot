# Настройка переменных окружения в Railway

## Пошаговая инструкция

### Шаг 1: Откройте настройки переменных окружения

1. В Railway Dashboard откройте ваш проект `accountant-bot`
2. Выберите сервис (service)
3. Перейдите на вкладку **"Variables"** (Переменные)

---

## Обязательные переменные окружения

### 1. `BOT_TOKEN`
**Что это:** Токен вашего Telegram бота

**Где взять:**
1. Откройте Telegram
2. Найдите бота [@BotFather](https://t.me/BotFather)
3. Отправьте команду `/mybots`
4. Выберите вашего бота
5. Нажмите "API Token"
6. Скопируйте токен (формат: `123456789:ABCdefGHIjklMNOpqrsTUVwxyz`)

**Как установить в Railway:**
- **Key:** `BOT_TOKEN`
- **Value:** Вставьте ваш токен (например: `8458288622:AAEqT2SEUgrseXlILTEINpoSv0YIEtvLGu0`)
- ⚠️ **Важно:** Это секретная информация! Railway автоматически скрывает значение

**Пример:**
```
BOT_TOKEN = 8458288622:AAEqT2SEUgrseXlILTEINpoSv0YIEtvLGu0
```

---

### 2. `BOT_USERNAME`
**Что это:** Username вашего Telegram бота (без @)

**Где взять:**
- Это имя вашего бота в Telegram (то, что идет после @)
- Например, если бот `@accountant_bot`, то username = `accountant_bot`

**Как установить в Railway:**
- **Key:** `BOT_USERNAME`
- **Value:** `accountant_bot`

**Статус:** ⚠️ **ОПЦИОНАЛЬНА** (есть дефолт)
**Дефолт:** `accountant_bot` (если не указано)
**Рекомендация:** Можно не указывать если username вашего бота = `accountant_bot`. Укажите только если username другой.

**Пример:**
```
BOT_USERNAME = accountant_bot
```

---

### 3. `DATA_DIR`
**Что это:** Путь к директории для хранения CSV файлов

**Важно:** Это должен быть путь, где смонтирован Volume в Railway!

**Как установить в Railway:**
- **Key:** `DATA_DIR`
- **Value:** `/data`

**Пример:**
```
DATA_DIR = /data
```

**⚠️ ВАЖНО:** Убедитесь, что в Railway создан Volume с mount path `/data`!

---

### 4. `KEEP_ALIVE_MODE`
**Что это:** Режим работы keep-alive для предотвращения sleep

**Как установить в Railway:**
- **Key:** `KEEP_ALIVE_MODE`
- **Value:** `always`

**Статус:** ⚠️ **ОПЦИОНАЛЬНА** (но рекомендуется для 24/7 работы)
**Дефолт:** `always` (если не указано)
**Рекомендация:** Рекомендуется установить `always` для гарантированной работы 24/7, но можно не указывать - дефолт уже `always`.

**Пример:**
```
KEEP_ALIVE_MODE = always
```

---

### 5. `PORT`
**Что это:** Порт для health check endpoint

**Как установить в Railway:**
- **Key:** `PORT`
- **Value:** `10000`

**Статус:** ⚠️ **ОПЦИОНАЛЬНА** - Railway обычно устанавливает автоматически
**Дефолт:** 10000 (если не указано)
**Рекомендация:** Можно не указывать - Railway установит автоматически. Но если хотите явно указать порт, установите `10000`.

**Пример:**
```
PORT = 10000
```

## Опциональные переменные (для Google Sheets)

**⚠️ ВАЖНО:** Бот поддерживает **несколько таблиц** - каждый пользователь может указать свою таблицу через команду `/setid` в боте. Переменная `GOOGLE_SPREADSHEET_ID` **НЕ обязательна** - она используется только как дефолтная таблица, если пользователь не указал свою.

### 6. `GOOGLE_CREDENTIALS` (Рекомендуется)
**Что это:** JSON содержимое credentials для Google Service Account

**Как получить:**
1. Перейдите в [Google Cloud Console](https://console.cloud.google.com)
2. Создайте проект (если нет)
3. Включите **Google Sheets API**
4. Перейдите в **IAM & Admin** → **Service Accounts**
5. Создайте новый Service Account или выберите существующий
6. Нажмите на Service Account → **Keys** → **Add Key** → **Create new key**
7. Выберите **JSON** формат
8. Скачайте файл
9. Откройте файл в текстовом редакторе
10. Скопируйте **весь JSON** (от `{` до `}`)

**Как установить в Railway:**
- **Key:** `GOOGLE_CREDENTIALS`
- **Value:** Вставьте весь JSON в одну строку
- ⚠️ **Важно:** JSON должен быть в одной строке! Удалите все переносы строк
- Используйте `${{ secret() }}` для безопасности (Railway предложит это автоматически)

**Пример формата:**
```
GOOGLE_CREDENTIALS = {"type":"service_account","project_id":"my-project","private_key_id":"...","private_key":"-----BEGIN PRIVATE KEY-----\n...\n-----END PRIVATE KEY-----\n","client_email":"...@....iam.gserviceaccount.com",...}
```

**Совет:** Если JSON большой, Railway может предложить использовать Secret. Нажмите на иконку `{}` и выберите "Create Secret" или используйте `${{ secret() }}`.

### 7. `GOOGLE_SHEET_NAME`
**Что это:** Имя листа в Google Таблице

**Как установить в Railway:**
- **Key:** `GOOGLE_SHEET_NAME`
- **Value:** `Sheet1` (или имя вашего листа)

**Статус:** ⚠️ **ОПЦИОНАЛЬНА** (есть дефолт)
**Дефолт:** `Sheet1` (если не указано)
**Рекомендация:** Обычно не требуется указывать, если лист называется `Sheet1`

**Пример:**
```
GOOGLE_SHEET_NAME = Sheet1
```

## Дополнительные переменные (не рекомендуются, только для особых случаев)

### 8. `GOOGLE_CREDENTIALS_PATH` ❌ НЕ РЕКОМЕНДУЕТСЯ (используйте GOOGLE_CREDENTIALS)
**Что это:** Путь к файлу credentials.json

**Статус:** ❌ **НЕ РЕКОМЕНДУЕТСЯ** для Railway
**Почему:** `GOOGLE_CREDENTIALS` (JSON в переменной) лучше работает на Railway
**Используйте только если:** Действительно нужно использовать файл вместо переменной

**Как установить в Railway:**
- **Key:** `GOOGLE_CREDENTIALS_PATH`
- **Value:** `/data/credentials.json`

**⚠️ Рекомендация:** Используйте `GOOGLE_CREDENTIALS` вместо этого!

---

### 9. `GOOGLE_SPREADSHEET_ID` ⚠️ ОПЦИОНАЛЬНА (обычно не требуется - пользователи сами устанавливают таблицы)
**Что это:** ID дефолтной Google Таблицы (если пользователи не указали свои)

**Статус:** ⚠️ **ОПЦИОНАЛЬНА** - используется только как fallback
**Логика:** Бот запоминает ID таблицы для каждого пользователя отдельно через `/setid`
**Рекомендация:** Обычно не требуется - пользователи сами устанавливают свои таблицы

**Как установить в Railway (если нужна дефолтная таблица):**
- **Key:** `GOOGLE_SPREADSHEET_ID`
- **Value:** ID таблицы из URL

**Пример:**
```
GOOGLE_SPREADSHEET_ID = 1BxiMVs0XRA5nFMdKvBdBZjgmUUqptlbs74OgvE2upms
```

---

### 10. `STORAGE_CSV_PATH` ❌ ИЗБЫТОЧНА (формируется автоматически из DATA_DIR)
**Что это:** Полный путь к CSV файлу для хранения расходов

**Статус:** ❌ **НЕ ТРЕБУЕТСЯ** - формируется автоматически из `DATA_DIR`
**Логика:** Путь формируется как `DATA_DIR + "/expenses.csv"` автоматически
**Рекомендация:** **Не устанавливайте** эту переменную - она избыточна. Используйте только `DATA_DIR`.

**Если все же нужно указать явно (не рекомендуется):**
- **Key:** `STORAGE_CSV_PATH`
- **Value:** `/data/expenses.csv`

**Пример (не рекомендуется):**
```
STORAGE_CSV_PATH = /data/expenses.csv
```

## Как добавить переменную в Railway

1. В разделе **Variables** нажмите кнопку **"Add"** (внизу справа)
2. В поле **Key** введите имя переменной (например, `BOT_TOKEN`)
3. В поле **Value** введите значение
4. Если значение секретное, Railway автоматически предложит использовать Secret
5. Нажмите галочку или Enter для сохранения

---

## Использование Secrets в Railway

Для секретных данных (токены, credentials) Railway предлагает использовать Secrets:

1. При вводе значения нажмите на иконку `{}`
2. Выберите **"Create Secret"** или используйте `${{ secret() }}`
3. Введите имя секрета
4. Введите значение
5. Railway автоматически подставит `${{ secret.SECRET_NAME }}`

**Пример:**
```
BOT_TOKEN = ${{ secret.BOT_TOKEN }}
GOOGLE_CREDENTIALS = ${{ secret.GOOGLE_CREDENTIALS }}
```

## Проверка настроек

После настройки всех переменных:

1. Сохраните изменения

   📁 Базовая директория данных: /data

## Частые ошибки

### Ошибка: "BOT_TOKEN не найден"
**Решение:** Проверьте, что переменная `BOT_TOKEN` установлена и значение правильное

### Ошибка: "CSV файлы теряются"
**Решение:** 
- Убедитесь, что `DATA_DIR=/data`
- Проверьте, что Volume создан и смонтирован в `/data`

### Ошибка: "Google Sheets не работает"
**Решение:**
- Проверьте `GOOGLE_CREDENTIALS` - JSON должен быть в одной строке
- Убедитесь, что Service Account имеет доступ к таблице пользователя
- Пользователь должен указать свою таблицу через `/setid` в боте
- `GOOGLE_SPREADSHEET_ID` не обязателен - используется только как fallback

## Как пользователи устанавливают свои таблицы

Бот поддерживает **несколько таблиц** - каждый пользователь может указать свою Google Таблицу:

### Шаг 1: Пользователь создает Google Таблицу
1. Пользователь создает новую Google Таблицу
2. Переименовывает лист на `Sheet1` (если нужно)

### Шаг 2: Пользователь дает доступ боту
1. В таблице нажимает "Share" (Поделиться)
2. Копирует email Service Account (пользователь может узнать его через команду `/email` в боте)
3. Вставляет email в поле "Add people and groups"
4. Выбирает роль: **Editor** (Редактор)
5. Нажимает "Send"

### Шаг 3: Пользователь указывает ID таблицы боту
**Способ 1: Команда `/setid`**
```
/setid 1BxiMVs0XRA5nFMdKvBdBZjgmUUqptlbs74OgvE2upms
```

**Способ 2: Просто отправить ID**
Пользователь просто отправляет ID таблицы боту (без команды)

**Где найти ID:**
- Открыть таблицу в браузере
- Посмотреть URL: `https://docs.google.com/spreadsheets/d/1ABC123.../edit`
- ID находится между `/d/` и `/edit`

### Шаг 4: Проверка
Пользователь отправляет боту:
```
расходы тест 100 руб
```

Если все настроено правильно, запись появится в Google Таблице пользователя.

### Где хранятся ID таблиц:
- В файле `/data/user_spreadsheet_ids.txt` (формат: `chatId=spreadsheetId`)
- Также синхронизируется с Google Sheets (лист "Users"), если настроен
